# 台股分析工具 — 功能說明

線上網址：<https://iamlchang-max.github.io/stock-tw/>

一個純前端（GitHub Pages）+ Google Apps Script 後端的台股分析工具，分成三個分頁：**圖表分析**、**買賣點警鐘**、**存股**。

---

## 架構總覽

| 部分 | 技術 | 說明 |
|---|---|---|
| 前端 | 原生 HTML / CSS / JS | 部署在 GitHub Pages，檔案：`index.html`、`app.css`、`app.js` |
| 後端 | Google Apps Script (`gas-code.gs`) | 存放警報／持股資料、Yahoo Finance 代理、每分鐘背景巡檢並發 Telegram 通知 |
| 圖表 | Lightweight Charts | K 線、均線、布林、MACD、RSI、成交量 |
| 行情來源 | Yahoo Finance | 透過自家 GAS proxy 取得（TWSE MIS 會擋 Google IP） |
| 資料儲存 | GAS ScriptProperties + 瀏覽器 localStorage | 警報／持股存後端；Telegram 設定、股票清單快取存本機 |

---

## 分頁一：📈 圖表分析

- **股票查詢繪圖**：輸入代號或用名稱搜尋，選擇月數（1～12），按「查詢 + 繪圖」。
- **即時資訊面板**（左側欄）：名稱、最新股價、開盤、最高、最低、昨收、成交量、漲跌（紅漲綠跌）。
- **技術圖表**：
  - K 線 + 移動平均線 + 布林通道
  - MACD
  - RSI
  - 成交量
- **名稱搜尋**：可用中文名稱或代號模糊搜尋（台股代號↔中文對照表，本機快取 24 小時）。
- **我的持股快選**（左側欄下方）：自動載入存股清單的股票，一鍵點擊即在本頁繪圖；新增／刪除持股會自動同步。

## 分頁二：🔔 買賣點警鐘

**目的**：對持股（或任意股票）設定買點／賣點價格，觸價時由後端發 Telegram 即時通知。

### Telegram Bot 設定
1. Telegram 搜尋 `@BotFather` → 傳 `/newbot` → 取得 Bot Token
2. 搜尋你建立的 Bot，傳任意訊息給它
3. 貼上 Token 後按「自動取得 Chat ID」→ 儲存 →「測試通知」驗證
   - ⚠ 背景監控由 GAS 負責（Token/Chat ID 已寫在 `gas-code.gs`）；此處設定僅供前端「測試通知」使用。

### 買賣點監控表
- **預設載入你的持股**（同代號去重）＋ 既有警報股，一檔一列。
- 每列欄位：代號、名稱、**現價**（紅漲綠跌）、**買點 ↓**、**賣點 ↑**、狀態、操作。
- **買點欄**填「跌到多少通知」；**賣點欄**填「漲到多少通知」，可只填一個或兩個都填。
- **儲存**：觸價時 GAS 每分鐘巡檢並發 Telegram 通知。改價後再儲存會自動「重新武裝」（即使先前已觸發）。
- **清除**：移除該檔的買賣點警報。
- **加入其他股票**：上方搜尋框可加入非持股的股票來監控。
- **狀態徽章**：買點／賣點分別顯示「監控中 / 已觸發（含觸發價與時間）/ 暫停」。
- 代號可點擊 → 跳到圖表分析頁繪圖。

> ⚠ 目前警報為**一次性**：某方向觸發後會停止，需重新儲存才再啟用（沿用後端原行為）。若要「每次觸價都通知、不自動停」需改 `gas-code.gs` 並重新部署。

## 分頁三：💰 存股

### 投資組合摘要（卡片）
- 總市值
- **總損益**（金額 ＋ 加權報酬率；賺紅虧綠）
- 年股息估算
- 平均殖利率
- 持股檔數

### 分布視覺化
- 市值佔比（圓餅圖）
- 年股息佔比（圓餅圖）

### 新增持股
欄位：代號／名稱（可搜尋）、股數、成本價／股（選填）、帳戶（如永豐、玉山）、標籤（逗號分隔，如「金融,高股息」）。

### 持股清單（表格）
- 欄位：代號、名稱、帳戶、標籤、持股、成本、現價、市值、**損益**、年股息、市場殖利率、個人殖利率、操作。
- **損益欄**：現價對比成本的盈虧，顯示金額 ＋ 百分比（賺紅虧綠；未填成本則顯示「—」）。
- **殖利率**：
  - 市場殖利率 = 年配息 ÷ 現價
  - 個人殖利率 = 年配息 ÷ 個人成本價
- **股利期間**可切換：近 12 個月 / 今年 / 去年。
- **標籤篩選**：點標籤晶片可過濾持股。
- **排序**：點欄位標題可依該欄排序（含損益）。
- **總計列**：加總股數、成本、市值、總損益、年股息，及加權殖利率。
- **每列操作**：看圖（跳圖表頁）、股數變更、成本、帳戶、標籤、刪除。

## 後端 `gas-code.gs` 重點

- **資料 API**：`doGet` 回傳 `{ alerts, holdings }`；`doPost` 支援動作：
  - 警報：`add` / `toggle` / `delete`
  - 持股：`addHolding` / `updateHolding` / `deleteHolding`
- **行情代理**：`?proxy=<url>`，僅允許白名單網域（TWSE、Yahoo Finance）。
- **背景巡檢**：`checkAlerts` 由時間觸發器每分鐘執行，僅在台股交易時段（週一~五 09:00~13:30）巡檢；觸價即發 Telegram 通知並標記 `triggered`。
- **輔助函式**：`resetAlerts`（清觸發狀態）、`setupTrigger`（建立每分鐘觸發器）。

### 部署注意
- 前端改動：`git push` 後 GitHub Pages 約 1～2 分鐘自動更新（瀏覽器按 `Ctrl+F5` 清快取）。
- 後端改動（`gas-code.gs`）：需手動貼回 Apps Script 專案並**重新部署**，前端 CRUD 用的是既有動作，一般不需動後端。
